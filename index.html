TECHNOLOGY

Our core product is MIA DB (multi-user interactive analytic database). By design it's in-memory scalable columnar OLAP database suitable for the cloud environments. There are minimum hardware requirements and it can run on any common x86-64 compatible server.


Low Latency & High Throughput

MIA DB uses the best features of the new generation OLAP databases e.g. lock-free and columnar architecture, compression, vector operations. More importantly it is optimized for high throughput and low latency which requires very different architecture. For this purpose, we combined concepts from outside world ( e.g. Git, Dropbox, Erlang ) with our previous experience in OLAP industry. E.g. on ordinary laptop, we can serve up to 1200 requests/sec.

Microcaching

Our main technological difference is automatic microcaching. It increases throughput for concurrent users several times comparing to traditional databases and allows to minimize the latencies for the vast majority of user requests. This is especially important for the usage in the cloud environments. And for the interactive user experience. Microcaching also slightly simplifies UI client and API by allowing the client to make synchronous requests only.

Flexible & Affordable

High throughput and low latency allow to simplify the overall architecture of the whole OLAP system. You can avoid custom layers of specialized caches which are necessary for fast system. Custom caches make the architecture complex, weaken the flexibility of the system and remarkably increases the total cost of the solution. MIA DB works on common hardware and decreases implementation and operating cost.
